# Indexing Service

هذه الخدمة مسؤولة عن بناء الفهارس (Indexes) التي يستخدمها محرك البحث للوصول إلى الوثائق بسرعة وفعالية دون الحاجة إلى البحث داخل جميع الوثائق في كل عملية استعلام.

الهدف من الفهرسة هو إنشاء تمثيل منظم للوثائق يمكن استخدامه لاحقاً أثناء عملية الاسترجاع (Retrieval) لتقليل زمن البحث وتحسين الأداء.

---

## الفهارس المنفذة حالياً

### 1. Inverted Index

يعتبر من أشهر هياكل البيانات المستخدمة في أنظمة استرجاع المعلومات.

فكرته الأساسية هي ربط كل مصطلح (Term) بالوثائق التي ظهر فيها مع عدد مرات ظهوره داخل كل وثيقة.

مثال:

```text
apple
→ [("Doc1", 2), ("Doc2", 1)]
```

هذا يعني أن كلمة:

```text
apple
```

ظهرت:

* مرتين في Doc1
* مرة واحدة في Doc2

يساعد هذا الفهرس على الوصول السريع إلى الوثائق ذات العلاقة دون الحاجة إلى فحص جميع الوثائق.

---

### 2. TF-IDF Index

يستخدم TF-IDF لتمثيل الوثائق عددياً داخل فضاء متجهي (Vector Space).

يعتمد على عاملين:

#### Term Frequency (TF)

عدد مرات ظهور الكلمة داخل الوثيقة.

#### Inverse Document Frequency (IDF)

مدى ندرة الكلمة ضمن كامل مجموعة الوثائق.

الفكرة هي إعطاء وزن أكبر للكلمات المهمة وتقليل تأثير الكلمات الشائعة.

قبل بناء TF-IDF يتم تطبيق:

```python
preprocess_document()
```

على جميع الوثائق لضمان استخدام نفس خطوات المعالجة المسبقة المستخدمة في بقية أجزاء النظام.

---

### 3. BM25 Index

يعتبر BM25 من أشهر نماذج الاسترجاع المعتمدة في محركات البحث الحديثة.

يعتمد على:

* عدد مرات ظهور الكلمة داخل الوثيقة
* طول الوثيقة
* ندرة الكلمة ضمن مجموعة الوثائق

ويتميز غالباً بأداء أفضل من TF-IDF في مهام الاسترجاع التقليدية.

يتم بناء BM25 باستخدام الوثائق بعد معالجتها مسبقاً بواسطة:

```python
preprocess_document()
```

ثم تحويلها إلى قائمة من الكلمات (Tokens).

## العلاقة مع Preprocessing Service

جميع الفهارس تعتمد على معالجة الوثائق قبل الفهرسة.

التدفق العام داخل النظام:

```text
Raw Documents
        ↓
Preprocessing Service
        ↓
Indexing Service
        ↓
Saved Index Artifacts
```

هذا يضمن أن الوثائق المستخدمة أثناء الفهرسة والاسترجاع تمر بنفس خطوات المعالجة المسبقة.

---

## اختيار مصطلحات الفهرسة

تم اعتماد المصطلحات الناتجة عن مرحلة المعالجة المسبقة (Preprocessing) كمصطلحات للفهرسة.

تشمل هذه المرحلة:

* Lowercasing
* Punctuation Removal
* Tokenization
* Stopword Removal
* Lemmatization

وبذلك يتم التخلص من الكلمات غير المهمة والاحتفاظ بالمصطلحات الأكثر فائدة أثناء عملية البحث.

## بناء الفهارس الفعلية

يتم بناء الفهارس المستخدمة في المشروع من خلال ملفات scripts.

TF-IDF:

```bash
python -m scripts.build_tfidf_index_sample
```

BM25:

```bash
python -m scripts.build_bm25_index_sample
```

تقوم هذه الملفات بقراءة الوثائق، تطبيق المعالجة المسبقة، بناء الفهرس، ثم حفظه داخل مجلد:

```text
artifacts/
```

لاستخدامه لاحقاً أثناء عملية البحث.
